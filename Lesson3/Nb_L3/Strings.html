STRINGS¶

String manipulation is arguably one of the most common activities in computer programming.
This is especially true in Web systems, where Java is a major player. In this chapter, we’ll look more deeply at what may be the most heavily used class in the language, String, along with some of its associated classes and utilities.

Conceptually, Strings are sequences of Unicode characters. For example string "Java\u2122" consists of five unicode characters "J", "a", "v", "a", "TM". Java doesn't have a built-in string type. Instead, the standard Java library has a class String. Each quoted string is an instance of class String.

String e = ""; // an empty String
String greeting = "Hello";
greeting.charAt(0);
Out[5]:

Substrings¶

There is a possibility to extract a substring from the bigger string. You should use substring() method of String class. For example,

In [7]:
String greeting = "Hello World";
String s = greeting.substring(0, 5);
System.out.println(s);
System.out.println(greeting);

Hello
Hello World

The second parameter of substring() is the first position that you do not want to copy. In our case, we want to copy positions 0, 1, 2(from position 0 to position 2 inclusive). As substring counts it it means from position 0 inclusive to position 3 exclusive.

Immutable Strings¶

Objects of the String class are immutable. If you examine the JDK documentation for the String class, you’ll see that every method in the class that appears to modify a String actually creates and returns a brand new String object containing the modification. The original String is left untouched.

In [2]:
public static String upcase(String s) {
    return s.toUpperCase();
}
    
String q = "JavaLang";
System.out.println(q);
String qq = upcase(q);
System.out.println(qq);
System.out.println(q);

JavaLang
JAVALANG
JavaLang

When q is passed in to upcase() it’s actually a copy of the reference to q. The object this reference is connected to stays in a single physical location. The references are copied as they are passed around. Looking at the definition for upcase() , notice that the vanishes. upcase() returns the result: a reference to the original String , with all the characters set to uppercase. But the reference it returns is for a new object, and the original q is left alone.

The String class gives no methods that let you change a character in a string. If you want to turn a string "Hello" to a string "Help!", you cannot directly change the last positions of a "Hello" string. To do such a thing you need to use create a new string. For example:

In [8]:
String greeting = "Hello";
String help = greeting.substring(0, 3) + "p!";
System.out.println(help);

Help!

Concatenation and StringBuilder¶

Java, like most others programming languages, allows you to use + to join (concatenate) to strings.

In [6]:

String world = "World";
String message = hello + " " + world;
System.out.println(message);

Hello World

This feature is commonly used in the output statements. For example: int age = 25; System.out.println("The person's age is " + a);

In [15]:
int a = 5;
int b = 10;
System.out.println("result = " + (a + b));

result = 15

If you need to put multiple strings together, separated by a delimiter, use the static join method:

In [19]:
String all = String.join(" / ", "S", "M", "L", "XL");
System.out.println(all);

S / M / L / XL

Since String objects are immutable, you can alias to a particular String as many times as you want. Because a String is read-only, there’s no possibility that one reference will change something that affect the other references. Immutability can have efficiency issues. A case in point is the operator + that is overloaded for String objects. Overloading means an operation has extra meaning when used with a particular class. (The + and += for String are the only operators overloaded in Java, and Java does not allow the programmer to overload any others.) The + operator concatenates String as:

In [1]:
String fruit = "Apple";
String thing = "Pen";
String result = thing + fruit + 100500;
System.out.println(result);

PenApple100500

Imagine how this might work. The String from variable thing could have a method append() that creates a new String object containing “Pen” concatenated with the contents of fruit. The new String object would then create another new String that added “100500” and so on. This would certainly work, but it requires the creation of many String objects just to put together this new String, then you have a bunch of intermediate String objects that must be garbage collected. But things no so bad because under the hood in the byte code you can find using of the StringBuilder. Here’s an example that produces a String result in two ways: using String s, and by hand-coding with StringBuilder:

In [3]:
public class DemoStringConcatenation {
    
    public String implicit(String[] fields) {
        String result = "";
        for (String s : fields) {
            result += s;
        }
        return result;
    }
    
    public String explicit(String[] fields) {
        StringBuilder result = new StringBuilder();
        for (String s : fields) {
            result.append(s);
        }
        return result.toString();
    }
}

The drawback of the first method that in the byte code in the loop StringBuilder object will be created for each iteration. In the second method we create StringBuilder explicitly. So, no penalties for our memory and performance. As a conslusion: prefer StringBuilder rather than +. With StringBuilder, String operations should be faster.

Testing Strings for Equality¶

To test whether two strings are equal, use the equals() method. The expression x.equals(y) returns true if the strings x and y are equal, false otherwise. Note, that x and y can be string literals or variables. For example:

In [2]:
String greeting = "Hello";
"Hello".equals(greeting);
Out[2]:
true
In [20]:
"Hello" == "Hello"
Out[20]:
true

To test where two string are equal except for the lower/uppercase distinction, use the equalsIgnoreCase() method

In [4]:
"Hello".equalsIgnoreCase("hello");
Out[4]:
true

Do not user the == operator to test whether two strings are equal. It only determines if two strins are stored in the same location in the memory. It is entirely possible to store same strings in different locations in the memory.

In [5]:
String s1 = "Ho-ho-ho";
if (s1.equals("Ho-ho-ho")) {
    System.out.println("Winter");
} else {
    System.out.println("Summer");
}

Winter

Everything seems to be OK. But here is an another case:

In [8]:
if (s1.substring(0, 2) == "Ho") {
    System.out.println("OK :)");
} else {
    System.out.println("NOT OK :(");
    System.out.println("s1.substring(0, 2) = " + s1.substring(0, 2));
}

NOT OK :(
s1.substring(0, 2) = Ho

Only string literals are shared. But strings, that are results of operations like substring or + are not. Therefore, never use == to compare strings.

Empty and Null Strings¶

The empty string "" is a string of the length 0. You can test whether the string is empty by calling if (str.length() == 0) or if (str.equals("")).However a String variable can also have a special value null. To test whether string is null use:
